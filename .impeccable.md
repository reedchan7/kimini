## Design Context

### Users

Kimini serves Kimi Code users who want the same coding-agent workflow in a native desktop client. They manage multiple workspaces and sessions throughout sustained development work.

### Brand Personality

Restrained, native, and tool-focused. The interface should feel calm, precise, and immediately familiar to existing Kimi Code Web users.

### Aesthetic Direction

Match Kimi Code Web 1:1 for visual hierarchy, interaction patterns, labels, ordering, progressive disclosure, and state feedback. Preserve Kimini's light and dark themes and its native GPUI rendering.

### Design Principles

1. Treat Kimi Code Web as the product and interaction reference.
2. Prefer direct, discoverable actions with secondary controls revealed on hover or focus.
3. Preserve native keyboard, focus, and screen-reader behavior.
4. Keep destructive actions explicit, confirmed, and visually distinct.
5. Reuse existing Kimini components and tokens when they can reproduce the reference faithfully.
